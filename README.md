# CAPUarticle

CAPUarticle 是一个用于撰写、润色、检查和整理学校宣传稿/学院新闻稿的 Codex skill。它面向新材料学院宣传场景，重点约束日期、事实、标题、规范用语、图片说明和投稿材料格式，尽量减少新闻稿中常见的事实遗漏和格式问题。

## 适用场景

- 根据活动信息、会议记录或素材草拟新闻稿。
- 润色已有学校推文、学院宣传稿、官网新闻动态。
- 检查稿件中的日期、标题、参与人员、领导称谓、政策术语等事实风险。
- 为投稿整理 Word 文档、图片命名和压缩包结构要求。
- 根据学院宣传部新闻稿撰写要求保持统一格式。

## 核心原则

- 先核实事实，再写作或修改。
- 不虚构日期、地点、人员、职务、活动环节和图片内容。
- 领导称谓不确定时先询问，不自行猜测。
- 新闻稿必须使用准确日期，避免使用“近日”等模糊表达。
- 图片按正文出现顺序命名为 `图1`、`图2`、`图3` 等。
- 投稿材料中，Word 文档和原始图片应放在同一目录，不单独放入图片子文件夹。

## 文件说明

- `SKILL.md`：skill 的主说明文件，定义触发场景、工作流程和输出要求。
- `references/writing-requirements.md`：新闻稿写作、排版、图片和投稿材料规则。
- `assets/新材料学院宣传部新闻稿撰写要求.docx`：原始参考文档，用于追溯规则来源。
- `agents/openai.yaml`：Codex/OpenAI 侧展示名称和默认提示配置。

## 使用方式

把该目录安装或放入 Codex skills 目录后，在需要写作或检查学校稿件时调用 `capuarticle`。例如：

```text
用 capuarticle 帮我润色这篇学院新闻稿
```

```text
用 capuarticle 根据这些活动信息写一篇官网新闻动态
```

```text
用 capuarticle 检查这份投稿材料还缺什么
```

## 投稿材料要求摘要

- Word 文档命名：`【YYYYMMDD】文章标题名称`
- 压缩包命名：与 Word 文档同名
- 原图命名：按正文顺序命名为 `图1`、`图2`、`图3`
- 目录结构：Word 文档和原图文件并列放在压缩包根目录
- 正文排版：正文宋体四号，首行缩进 2 字符，1.5 倍行距
- 小标题排版：宋体四号，加粗，左对齐，不首行缩进
- 图片说明：宋体五号，居中，简洁准确

## 注意

该 skill 只根据用户提供或可验证的材料处理稿件。若活动时间、人员身份、审核信息、图片内容等事实不明确，应先补充确认后再声称稿件可以提交。
